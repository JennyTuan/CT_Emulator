# Product Requirement Documents (PRD)

此目录用于存放 CT Emulator 项目中各个功能模块的产品需求文档（PRD）。

## 文档规范

每一个功能模块应包含独立的 `.md` 文件，描述以下内容：

- **功能概述**：该模块的核心目的。
- **用户角色**：使用此功能的具体角色（如：医生、维修工程师）。
- **业务流程**：核心的操作逻辑。
- **界面原型与交互**：UI 布局要求与交互细节。
- **数据项说明**：相关的状态变量与后端接口字段映射。
